# Release Readiness Review - 2026-05-28

## Verdict

`0.1.0-beta.1` has passed the bounded release-prep verification gate.

The release-prep work is now a beta-readiness gate instead of metadata-only
cleanup. The main blockers from the initial review have been converted into
code changes and contract checks.

## Resolved Beta Blockers

### Runtime Artifact Contamination

- Desktop packaging now uses a curated `agentin` filter instead of raw
  `agentin/**/*`.
- A2A task/event logs, timestamped sub-agent verifier folders, runtime memory,
  workspaces, user data, caches, generated outputs, Python bytecode, logs, and
  plugin runtime state are excluded.
- `.gitignore` and `.dockerignore` now reject A2A runtime data, timestamped
  sub-agent verifier folders, and Python bytecode.
- `tests/contracts/release-runtime-artifacts-contract.test.js` fails if tracked
  or packaged paths include runtime artifacts.

### Remote Gateway Secret Storage

- Remote Gateway shared secret now persists through
  `db.setCredential('remoteGateway.secret', value)`.
- Reads use `db.getCredential('remoteGateway.secret')`.
- Migration covers both legacy plaintext `remoteGateway.secret` settings and
  existing generic `setting.remoteGateway.secret` credentials.
- Contract coverage was added beside the generic secret migration guard and the
  Remote Gateway contract.

### Contract Drift

- The Companion Browser pairing contract now matches the current
  `client.createChatSession()` / `ensureActiveSession()` flow.
- The Pixel Avatar contract now matches the preset-based character and emotion
  configuration model.

### Line Budget

- Pixel Avatar rendering code was split into:
  - `avatar.js`
  - `avatar-common.js`
  - `avatar-cat.js`
  - `avatar-robot.js`
  - `avatar-girl.js`
- The temporary line-budget allowlist entry for `avatar.js` was removed.

### Direct Correctness Fixes

- `PortListenerManager` no longer emits the invalid `http://localhost:*` CORS
  origin and clears body-parser timers.
- `addPromptRule()` returns the inserted DB row instead of a hardcoded active
  flag.
- `deleteAgent()` now cleans associated sessions/state and removes the owned
  agent folder.
- Dead helpers, redundant requires, and debug logging were removed.

### Docs And Metadata

- Root package, Wellbot package, and mobile metadata use `0.1.0-beta.1`.
- README beta/security/npm wording now matches the implementation.
- Wellbot package metadata includes `repository` and `homepage`.
- Wellbot README identifies the beta CLI surface.

## Release Gates

  `bin/wellbot.js`, and `package.json`.

The sandbox returned `spawn EPERM` for `quick`; the same bounded suite was
rerun outside the sandbox and passed. `core` was run outside the sandbox for the
same child-process requirement.

## Remaining Non-Blocking Debt

The following should not block `0.1.0-beta.1`, but should stay visible before a
broader beta:

- Electron still runs with `sandbox: false`.
- Remote Gateway still needs status redaction and session-management follow-up.
- Tool pattern stripping is duplicated.
- Inference dispatcher lock state can race.
- Several renderer/main files are near the 1000-line budget.
- Test coverage has known gaps around destructive cleanup, mobile transport,
  and companion token lifecycle.

The longer list lives in `docs/architectural-debt.md`.
